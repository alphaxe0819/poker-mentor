⑥ 新電腦加入（統一流程）

不管是第 2 台還是第 10 台，每台新電腦都跑同樣的 5 個步驟就能接入雲端開發。

新電腦設定 SOP

  • 1
    Clone 專案 git clone https://github.com/alphaxe0819/poker-mentor.git
  • 2
    切換到目前的工作分支 git checkout feature/xxx 或 dev，看當前在開發什麼
  • 3
    安裝 dependencies npm install
  • 4
    自動產生環境變數 powershell scripts/setup-env.ps1 Windows 用 .ps1，Mac/Linux 用 bash scripts/setup-env.sh
  • 5
    安裝 Claude Code npm install -g @anthropic-ai/claude-code 登入後即可開始開發

⑦ 注意事項

開發守則

✅
每完成一組邏輯完整的改動就立即 commit，不要累積
✅
每次開工、換機前一定先 git pull
✅
收工前一定 git push，確保雲端有最新版本
✅
.env 每台電腦要一致，改了記得同步所有機器
✅
先 merge 到 dev 測試，確認沒問題才 merge 到 main
✅
DB migration 先在測試 Supabase 跑過，再跑正式的
❌
不要在有大量未 commit 改動時切換分支（會丟失工作）
❌
不要跳過 dev 直接 merge feature 到 main
❌
不要把 .env 或 node_modules commit 進 Git
❌
不要在多台同時改同一個檔案（容易 merge conflict）

環境同步清單

項目 Git 同步？ 每台新機需手動處理？
原始碼 src/ ✅ 自動 —
package.json ✅ 自動 跑 npm install
.env ❌ 不進 Git 跑 scripts/setup-env.ps1 自動產生
node_modules/ ❌ 不進 Git npm install 生成
Supabase 正式設定 ❌ 不進 Git 寫在 .env + Vercel 正式專案 env vars
Supabase 測試設定 ❌ 不進 Git 寫在 .env.staging + Vercel 測試專案 env vars
